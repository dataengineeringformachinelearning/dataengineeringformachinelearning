Authentication & Access

Secure access to our high-throughput pipelines requires an API key. Register an account, generate a secure token in your profile, and pass it in the Authorization header.

HTTP Header
Authorization: Bearer YOUR_API_KEY

Data Ingestion

High-throughput endpoint optimized for receiving batched or streaming data from distributed pipelines like Apache Spark or Databricks.

POST /api/v1/ingest
cURL Request
curl -X POST https://backend.deml.app/api/v1/ingest \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{{ '{' }}"batch_id": "123", "records": [{{ '{' }}"feature1": 1.0{{ '}' }}]{{ '}' }}'

Real-time Inference

Low-latency endpoint for model inference, ideal for Kubernetes sidecars or deep learning services calling models directly.

POST /api/v1/predict
cURL Request
curl -X POST https://backend.deml.app/api/v1/predict \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{{ '{' }}"model_version": "v2", "inputs": [0.5, 0.2]{{ '}' }}'

  • Apache Spark: Write DataFrames directly to ingest.
security

Enterprise Security

All tenant data is cryptographically isolated using strict multi-tenancy rules. API keys are shown once at creation and hashed with SHA-256 at rest. Infrastructure uses Google Distroless multi-stage container builds to eliminate shell-based attack surfaces.
